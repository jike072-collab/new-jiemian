# 奥皇 AI 工作区参考拆解规格

## 使用边界

这份文档只学习参考站的工具工作区结构、布局比例、组件样式、按钮状态、细节效果和交互方式。  
不复制对方品牌、Logo、图片、专有文案或代码。奥皇 AI 使用自己的品牌、自己的功能和自己的结果展示。

本阶段不学习营销落地页内容，不做 FAQ、卖点介绍、使用步骤、博客、订阅拉新、营销大图。  
参考对象只限于工具工作区：左侧功能导航、中间参数面板、右侧结果/引导工作区。

参考来源：

- 参考站工具页：https://aivideomaker.ai/zh/text-to-image-z-image-turbo
- 用户提供的参考截图和要求
- 当前项目页面结构：`src/components/studio-app.tsx`、`src/app/globals.css`
- `product-design` 评审子线程反馈

## 一、页面骨架拆解

### 1. 左侧功能导航区

参考工作区左侧是固定导航，不是大卡片区。

应采用：

- 固定在页面最左侧。
- 深黑背景，和页面背景区分轻微即可。
- Logo 在顶部，品牌识别简洁，不做大宣传。
- 菜单是小图标 + 文本的紧凑列表。
- 不要大卡片按钮，不要每一项都带长描述。
- 当前激活菜单项有细描边、浅色底、左侧色条或轻高亮。
- 分组标题是小号灰色文字，例如“生成”“增强”“管理”。
- NEW / 订阅 / 积分 标签是小胶囊，不能抢菜单主文字。
- 设置、登录、后台入口放底部，低权重呈现。

建议尺寸：

- Sidebar 宽度：桌面 240px 到 260px，当前可保持 250px。
- 顶部品牌区高度：56px 到 72px。
- 分组标题字号：11px 到 12px。
- NavItem 高度：40px 到 44px。
- 图标：16px 到 18px。
- 菜单内边距：8px 到 12px。

### 2. 中间参数面板

参考工作区中间是窄控制台，不是宽表单，也不是多张大卡片堆叠。

应采用：

- 位于左侧导航右边。
- 宽度约 360px 到 420px，当前 380px 合理。
- 深灰卡片背景，边界明确。
- 内部包含标题、模型选择、上传框、提示词、参数选择、生成按钮。
- 表单从上到下连续填写，节奏紧凑。
- 分组可以存在，但要轻，不能每段都像独立营销卡片。
- 提示词是核心动作，应比参数更突出。
- 必填项在标签右侧显示红色 `*`。
- 积分提示可预留在清晰度或生成按钮附近，但不要变成宣传块。

推荐顺序：

1. 工具标题和一句说明。
2. 模型选择。
3. 参考图片/视频上传，可选时显示“可选”。
4. 提示词输入。
5. 比例、时长、清晰度等参数。
6. 生成按钮和所需积分提示。

### 3. 右侧工作区

参考工作区右侧是结果区和引导区，不是居中的内容页。

应采用：

- 占据左侧导航和中间面板之外的全部剩余宽度。
- 占据页面全部剩余高度。
- 默认显示引导态，不默认显示大结果图。
- 生成中显示等待态。
- 生成后切换结果态。
- 右侧必须撑满剩余空间，不允许左右大空白。
- 内部可以有左右分栏，但这是 Workspace 内部分栏，不是页面外层居中。

右侧三种状态：

- 默认引导态：告诉用户“结果会显示在这里”，提示完成输入、选择参数、点击生成。
- 生成中态：显示任务正在处理、轻量加载动画、当前任务类型。
- 结果态：图片/视频成为主视觉，下方放下载、重新生成、保存、查看历史等动作。

## 二、设计 Token 拆解

以下是参考工作区气质的近似 Token，不是复制色值。奥皇 AI 可以在此基础上保留自己的品牌色，但应控制使用范围。

| Token | 建议值 | 使用说明 |
| --- | --- | --- |
| 页面背景色 | `#050505` 到 `#080808` | 全屏工作区底色，深黑、克制。 |
| 左侧导航背景色 | `#0A0A0B` 到 `#101010` | 与页面背景轻微分层。 |
| 中间面板背景色 | `#111111` 到 `#161616` | 窄控制台主体。 |
| 右侧工作区背景色 | `#0D0D0E` 到 `#141414` | 结果和引导容器。 |
| 边框色 | `rgba(255,255,255,0.08)` 到 `rgba(255,255,255,0.14)` | 大部分描边使用低对比。 |
| 主文字色 | `rgba(255,255,255,0.92)` 到 `#FFFFFF` | 标题、按钮、当前项。 |
| 次文字色 | `rgba(255,255,255,0.52)` 到 `rgba(255,255,255,0.68)` | 描述、说明、标签。 |
| 弱文字色 | `rgba(255,255,255,0.34)` 到 `rgba(255,255,255,0.45)` | 分组标题、辅助信息。 |
| 强调色 | 品牌紫/粉，建议 `#D946EF` 或更克制同系色 | 只用于主按钮、焦点态、当前项。 |
| 按钮主色 | `#E83E8C` / `#D946EF` / 品牌渐变低强度版 | 不要全页面泛紫。 |
| hover 状态 | 背景提高 3%-6% 亮度，边框变亮 | 不做大幅上浮。 |
| active 状态 | 轻微压下或高亮边框 | 用于点击反馈。 |
| disabled 状态 | 背景灰、文字弱、鼠标不可点 | 同时显示不能生成的原因。 |
| 输入框背景 | `#171717` 到 `#1D1D1D` | 与面板背景区分轻微。 |
| 上传框虚线颜色 | `rgba(255,255,255,0.18)` | hover 可提高到 0.28。 |
| 圆角大小 | 主面板 14px-18px；控件 10px-14px；胶囊 999px | 当前过大的 2rem 圆角需要收。 |
| 卡片间距 | 外层 12px-16px；表单项 12px；组内 8px-10px | 工作区要紧凑。 |
| 字号层级 | 页面标题 20px-24px；面板标题 18px-20px；标签 13px-14px；说明 12px-13px | 避免英雄式大标题。 |

## 四、当前页面差异

### 左侧导航为什么不像

- 当前导航项更像大卡片按钮：有大圆角、描述文字、大渐变选中态，视觉重量过高。
- 参考站的左侧更像紧凑工具列表，用户可以快速扫功能。
- 当前有提示卡和统计块，会让左侧像后台首页，而不是工具导航。
- 后台设置、客户登录和核心工具入口视觉等级接近，主次不够明确。

### 中间表单为什么不像

- 当前中间列宽度对，但内部每段都包成大 FormSection，像步骤卡片。
- 参考站更像一条连续控制台流程，控件密度更高。
- 当前标题、编号、说明过多，填写路径被切碎。
- 提示词不应该被厚重分组包住，它应是最核心输入。
- “AI Tool”“积分位预留”“客户登录入口”偏内部占位，不像正式工作区。

### 右侧工作区为什么不像

- 当前默认态有大图和多张说明卡，容易像教程/展示页。
- 参考工作区右侧应先是轻引导，结果生成后才切换真实预览。
- 右侧应强调“这里等待结果”，而不是默认展示一张大成品图。
- 生成中、失败、成功三种状态需要更清楚。

### 按钮为什么不像

- 当前按钮和菜单有较多渐变、阴影、上浮效果，偏展示型。
- 参考工具按钮更像操作控件：明确、紧凑、状态清楚。
- disabled 状态需要说明原因，例如未填提示词、未选模型、未上传必需文件。
- 主按钮应尽量整宽，文案短而直接。

### 空白为什么多

- 之前页面像官网内容页时使用过居中容器、大 padding、卡片内卡片。
- 当前三列骨架已接近，但右侧默认内容仍有“展示页式留白”。
- 右侧 Workspace 应填满剩余空间，内部空态也要撑满，不应把内容缩成居中说明块。

### 紫色玻璃风为什么不接近

- 参考站工具区整体深黑、克制，强调色用于按钮和选中态。
- 当前粒子、彩色径向光、玻璃阴影、紫色渐变较多，容易变成营销视觉。
- 工具区需要稳定和可读，不需要大面积氛围装饰。

### 结果区为什么不应该默认显示大图

- 用户还没生成时，大图会误导用户以为已有结果。
- 默认态应该让用户知道下一步做什么，而不是浏览示例。
- 示例可以弱化成小提示或空预览占位。
- 生成后真实结果才应该成为右侧主视觉。

### 哪些状态块应该删除或弱化

- 左侧“左边选功能，中间填信息，右边看结果”提示卡。
- 图片/视频入口数量统计块。
- 中间顶部“积分位预留”“客户登录入口”。
- 右侧默认态的大图教程卡片堆。
- 大面积粒子、彩色径向光、面板 hover 上浮。
- 与当前生成任务无关的后台、账号、未来能力提示。

## 五、实施计划

### Phase 1：只改整体三列布局和空白问题

目标：

- 锁定桌面三列：Sidebar 250px / ToolPanel 380px / Workspace 剩余宽度。
- 删除主工作区中的 `max-width`、`margin auto`、官网容器。
- 确保 `html`、`body`、根节点、AppShell 全屏。
- 确保右侧 Workspace 和内部状态卡都是 100% 宽高。
- 大屏不居中变窄，小屏再按规则压缩。

验收：

- 1920px、1440px、1280px、1180px 下无左右大空白。
- 页面没有横向滚动条。
- 右侧始终填满剩余空间。

### Phase 2：只改左侧导航

目标：

- 把大卡片导航改成紧凑列表。
- 删除左侧提示卡和统计块。
- 增加分组标题。
- 当前项使用细高亮，而不是大渐变。
- 后台设置、客户登录、语言/积分入口放到底部低权重区。

验收：

- 左侧一眼是“工具菜单”，不是后台首页。
- 菜单可以快速扫，不需要读长描述。

### Phase 3：只改中间参数面板

目标：

- 保持 360px-420px 窄控制台。
- 表单顺序调整为：标题、模型、上传、提示词、参数、生成。
- 弱化或删除厚重 FormSection。
- 统一标签、必填星号、提示文案。
- 上传框、提示词、比例、清晰度控件更紧凑。

验收：

- 用户从上到下能自然完成一次生成。
- 没有内部占位文案。
- 必填项清楚，非技术用户看得懂。

### Phase 4：只改右侧引导态 / 结果态

目标：

- 默认引导态不显示大结果图。
- 生成中显示轻量等待态。
- 结果态显示真实图片/视频，并提供结果动作。
- 失败态显示清楚原因和重试入口。

验收：

- 未生成时不会误以为已有结果。
- 生成后无需滚动即可在右侧看到预览。
- 右侧状态始终占满空间。

### Phase 5：只改按钮、输入框、上传框、hover/active/disabled 细节

目标：

- 主按钮整宽或接近整宽，状态完整。
- 输入框、下拉、上传框统一深灰背景和 focus 边框。
- RatioSelector 从大卡片变为小比例按钮。
- hover 只做轻背景/边框变化。
- active 轻压下即可。
- disabled 灰化并显示原因。

验收：

- 每个可点控件都知道“默认、悬停、选中、不可用”是什么样。
- 不再依赖大阴影、大渐变、大漂浮制造质感。

### Phase 6：截图对比并返工

目标：

- 用参考站工作区截图和奥皇 AI 当前页做并排检查。
- 分别检查桌面 1920px、1440px、1280px、1180px。
- 检查左侧密度、中间表单节奏、右侧状态、按钮状态。
- 根据截图只返工差距最大的部分。

验收：

- 页面第一眼是 AI 工具工作区，不是官网内容页。
- 三列清楚，右侧不空。
- 控件形态、按钮状态、细节效果接近参考站气质。
- 没有营销内容回流。
